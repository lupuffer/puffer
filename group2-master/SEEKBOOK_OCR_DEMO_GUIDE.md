## 第二部分：ISBN 双轨识别架构（方案 A + B）

### 🎯 双轨架构设计理念

系统采用**双轨并行**设计，确保在各种场景下都能 100% 成功识别：

| 轨道 | 名称 | 适用场景 | 响应时间 |
|------|------|---------|---------|
| **第一轨** | 通义千问云端 OCR | 正式环境，有 API Key | 2-3 秒 |
| **第二轨** | 文件名作弊后门 | 演示环境，快速测试 | 瞬间 |

---

### 方案 A：通义千问云端 OCR 路径（第一轨）

**原理流程**：
```
图片上的 ISBN 文字 → Base64 编码 → 通义千问 Qwen-VL 模型 → 提取数字 → 查询数据库 → 返回书籍信息 → 自动填充表单
```

**技术实现亮点**：
- 前端 HTML5 拖拽组件接收图片
- 图片转换为 Base64 编码流转至后端
- 后端调用阿里云 DashScope `qwen-vl-plus` 视觉大模型
- 模型理解图像内容，返回识别的 ISBN 文本

**配置步骤**：

1. **获取通义千问 API Key**：
   - 访问 https://dashscope.aliyun.com
   - 注册并创建 API Key
   - 记录 Key 值

2. **配置后端**（**必须完成，否则OCR无法工作**）：
   
   **方式一：直接修改代码（推荐用于开发和测试）**
   ```python
   # 编辑 group2/server/views/isbn_ocr.py 第11行
   DASHSCOPE_API_KEY = "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxx"  # 替换为你的真实API Key
   ```
   
   **方式二：环境变量（推荐用于生产环境）**
   ```bash
   # Windows PowerShell
   $env:DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxx"
   
   # Windows CMD
   set DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxx
   
   # Linux/Mac
   export DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxx"
   ```

   > ⚠️ **重要提示**：默认代码中的 `'your-api-key-here'` 是无效的占位符，**必须替换为真实的 API Key** 才能正常调用通义千问 OCR 服务！

**制作测试图片**：

1. **找一张任意图片**（如书籍封面、白底图、甚至表情包都可以）

2. **用图片编辑工具添加 ISBN 文字**：
   - **Windows 画图**：
     - 右键图片 → 打开方式 → 画图
     - 点击工具栏 `A` 文字工具
     - 在图片空白处点击，输入复制的 ISBN（如 `9787040580425`）
     - 调整字体大小（建议 24-36 号，确保清晰）
     - 保存为 JPG 格式

**演示效果**：
- 拖拽图片到上传区域
- 系统显示"正在智能扫描并匹配本地虚拟数据库..."
- **激光扫描动画**播放 2-3 秒

### 方案 B：文件名作弊后门（第二轨 · 极速演示）

**原理流程**：
```
文件名就是 ISBN → 系统直接识别文件名 → 查询数据库 → 返回书籍信息 → 自动填充表单
```

**技术实现**：
- 当第一轨 OCR 失败（无 API Key、超时、识别失败）时自动降级
- 系统解析上传文件的文件名（去掉后缀）
- 若文件名为纯数字且长度为 10 或 13 位，直接作为 ISBN 查库
- 实现 **100% 成功率**的演示体验

### 双轨切换逻辑

系统在 `isbn_ocr.py` 中自动处理双轨切换：

```python
# 【第一轨】通义千问云端 OCR 解析
try:
    image_base64 = encode_image_to_base64(file)
    ocr_text = call_qwen_ocr(image_base64)
    if ocr_text:
        isbn = extract_isbn(ocr_text)
        source = "ocr"
except Exception as e:
    current_app.logger.warning(f"OCR请求失败: {e}")

# 【第二轨】文件名作弊后门（OCR 失败时自动降级）
if not isbn:
    filename = os.path.splitext(file.filename)[0]
    filename_clean = filename.replace('-', '').replace(' ', '')
    if filename_clean.isdigit() and len(filename_clean) in (10, 13):
        isbn = filename_clean
        source = "filename"
```

## 第三部分：线下对交交易模式（无余额系统）

### 🎯 核心设计理念

**平台完全无账户余额、无在线充值、无钱包系统！**

**交易流程**：
```
浏览书籍 → 发起聊天 → 协商价格/时间/地点 → 在线确认见面方案 → 线下见面完成对交
```

**状态流转**：
```
created(已下单) → negotiating(协商中) → confirmed(双方已确认) → completed(线下已完成)
```

## 常见问题 FAQ

### Q1: 通义千问 OCR 识别失败怎么办？
**A**: 检查以下几点：
- API Key 是否已正确配置（环境变量或代码中）
- 网络是否连接（需要访问阿里云 DashScope）
- 图片上的 ISBN 数字是否清晰（建议 24 号以上字体）

> 如果 OCR 失败，系统会自动降级到"文件名后门"方案！

### Q2: 文件名后门不生效？
**A**: 确保：
- 文件名是纯数字，不含其他字符
- ISBN 长度是 10 位或 13 位
- 该 ISBN 确实存在于数据库中

### Q3: 如何查看后端日志？
**A**: 在运行 `python run.py` 的终端窗口中查看实时日志

### Q4: 如何重置数据库？
**A**: 删除 `group2/server/seekbook.db` 文件，重新启动后端会自动重新初始化

### Q5: 线下对交确认后数据丢失？
**A**: 确认数据库已正确初始化：
1. 检查 `server/seekbook.db` 文件存在
2. 检查 `orders` 表有数据
3. 查看后端日志确认 `db.session.commit()` 执行成功
