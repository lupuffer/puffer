## 四、线下交易模式设计（符合需求）

### 4.1 Order模型设计

```python
class Order(db.Model):
    # 线下交易状态: created -> confirmed -> completed
    status = db.Column(db.String(20), default='created')
    
    # 线下见面信息（核心字段）
    meet_time = db.Column(db.String(100))   # "2026-05-30 14:00"
    meet_place = db.Column(db.String(200))  # "紫金港校区东大门"
    
    # 无balance/wallet字段 ✅
```

### 4.2 订单状态流转

```
已下单(created) → 双方确认见面方案(confirmed) → 线下交付完成(completed)
```

**✅ 无在线支付、无钱包余额、无资金流转**，完全符合"线上沟通确认线下交易地点"需求。

## 五、ISBN识别 - 通义千问API预留

```python
# views/isbn_ocr.py
OCR_API_URL = "https://api.ocr.example.com/recognize"  # 预留接口

# 用户只需替换为通义千问实际地址：
# OCR_API_URL = "https://dashscope.aliyuncs.com/api/v1/..."
```

**双轨制设计**:
1. 第一轨：调用OCR_API_URL识别图片中的ISBN
2. 第二轨：OCR失败时，使用文件名作为ISBN
3. 最终都严格查数据库 `db.session.query(Book).filter_by(isbn=isbn).first()`
