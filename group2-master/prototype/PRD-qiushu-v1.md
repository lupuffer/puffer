## 2. 技术实现说明

### 2.1 技术约束（已满足）
- ✅ 纯前端实现，无框架（无Vue/React）
- ✅ 无组件库（无Ant Design/Element UI）
- ✅ 仅使用基础HTML、CSS、JavaScript
- ✅ 使用Promise加载数据，无回调函数
- ✅ 代码中无`var`，全部使用`const`和`let`
- ✅ 仅使用有限DOM类型：div, p, h1-h6, span, a, img, ul, ol, dl, input, i

### 2.2 后端架构（200行规范铁律）

系统后端采用 **Flask Blueprint 工厂化架构**，严格遵循 **"任何后端单文件代码量不得超过 200 行"** 的开发铁律：

```
server/
├── app.py              # 应用工厂（仅负责初始化与蓝图挂载）≤ 60行
├── config.py           # 配置中心 ≤ 50行
├── models.py           # 数据库模型定义 ≤ 150行
├── init_data.py        # 十万级数据初始化 ≤ 180行
├── views/              # 蓝图视图目录（高内聚低耦合）
│   ├── auth.py         # 用户认证蓝图 ≤ 100行
│   ├── books.py        # 书籍管理蓝图 ≤ 150行
│   ├── chats.py        # 即时通讯蓝图 ≤ 120行
│   ├── orders.py       # 线下对交蓝图 ≤ 180行
│   └── isbn_ocr.py     # ISBN双轨识别蓝图 ≤ 150行
└── uploads/            # 文件上传持久化目录
```

- **统一的按钮样式**：`.btn`, `.btn-primary`, `.btn-secondary` 等类名统一

## 3. 交易模式：线下对交（无余额系统）

### 3.1 核心设计理念

**平台完全无账户余额、无在线充值、无钱包系统、无线上支付！**

### 3.2 交易流程

```
浏览书籍 → 发起聊天 → 协商价格/时间/地点 
    ↓
在线确认见面方案（数据持久化至SQLite）
    ↓
线下见面完成对交
```

### 3.3 订单状态流转

| 状态 | 说明 | 操作 |
|------|------|------|
| `created` | 已下单 | 买家发起咨询，系统自动创建订单 |
| `negotiating` | 协商中 | 双方聊天协商价格、时间、地点 |
| `confirmed` | 双方已确认 | 任一方点击"确认线下对交方案" |
| `completed` | 线下已完成 | 见面交易后任一方标记完成 |
| `cancelled` | 已取消 | 交易取消 |

### 3.4 数据持久化字段

```python
# orders 表核心字段
meet_time = db.Column(db.String(100))      # 见面时间（如"2026-05-30 14:00"）
meet_place = db.Column(db.String(200))     # 见面地点（如"紫金港校区东大门"）
confirmed_at = db.Column(db.DateTime)      # 确认时间戳
status = db.Column(db.String(20))          # 订单状态
```

### 3.5 强持久化验证

- 确认见面方案后，数据立即写入 SQLite
- 刷新页面（F5）后，见面信息依然显示
- 退出浏览器重新登录，数据完整保留
- 可在 VSCode SQLite Viewer 中查看确认

## 6. ISBN 双轨识别架构（新增核心功能）

### 6.1 双轨设计理念

| 轨道 | 名称 | 适用场景 | 响应时间 |
|------|------|---------|---------|
| **第一轨** | 通义千问云端 OCR | 正式环境，有 API Key | 2-3 秒 |
| **第二轨** | 文件名作弊后门 | 演示环境，快速测试 | 瞬间 |

### 6.2 第一轨：通义千问云端 OCR

**流程**：
```
图片 → Base64编码 → 通义千问 Qwen-VL → 提取ISBN → 查询数据库 → 自动填表
```

**配置**：
```python
# server/views/isbn_ocr.py
DASHSCOPE_API_KEY = "your-api-key-here"  # 或环境变量
DASHSCOPE_MODEL = "qwen-vl-plus"
```

### 6.3 第二轨：文件名后门

**流程**：
```
文件名（如9787040580425.jpg）→ 解析ISBN → 查询数据库 → 自动填表
```

**触发条件**：
- 第一轨失败（无API Key、超时、识别失败）
- 文件名为纯数字且长度为10或13位

### 6.4 前端识别组件

```html
<!-- ISBNBarcodeUploader.vue -->
<div class="drop-zone" @drop="handleDrop">
    <div class="scanner-effect">
        <div class="laser-line"></div>
    </div>
    <p>拖拽图片识别ISBN</p>
</div>
```
