## 🚀 启动准备（演示前必做）

### 1. 启动后端
```bash
cd server
pip install -r requirements.txt
python run.py
```
✅ 看到输出 `SeekBook Flask 后端服务启动中... API 地址: http://127.0.0.1:5000/api` 即成功。

### 2. 启动前端
```bash
cd client
npm install
npm run dev
```
✅ 看到输出 `http://localhost:5173/` 即成功。  
✅ 浏览器打开 `http://localhost:5173/`。

### 3. 初始化数据库
后端启动时会自动创建表并初始化数据。  
也可以手动调用 `POST /api/init`。

## ⭐ 15-20 分钟推荐展示路线

### 展示主线：从新用户进入平台到完成一次校园二手书流转

| 时间 | 展示段落 | 页面/操作 | 讲解重点 |
|------|----------|-----------|----------|
| 1-3 分钟 | 现场注册与登录校验 | `/login`：注册新用户；先输错密码登录失败，再用正确密码登录 | 满足“注册、登录现场操作”和“密码错误登录失败”要求 |
| 3-5 分钟 | 首页快速总览 | `/`：看书单进度、市场看板、信誉、知识推荐 | 说明平台不是单一买卖页，而是交易 + 学习经验的校园闭环 |
| 5-8 分钟 | 买书主流程 | `/buy` 搜索/筛选 → 书籍详情 → 收藏/私聊/立即购买 | 展示商品发现、卖家信誉、订单创建 |
| 8-11 分钟 | 分支流程 | 缺货登记；或订单取消；或未登录时点击收藏/购买触发登录提示 | 用“异常/分支”替代投诉流程。当前系统没有独立投诉模块，不建议现场硬讲投诉 |
| 11-14 分钟 | 卖书主流程 | `/sell`：ISBN 识别 → 保存草稿 → 发布 | 强调 OCR 自动填表、草稿未发布不算商品、发布后才进入市场 |
| 14-16 分钟 | 交易履约 | `/messages` 或 `/orders`：聊天协商、确认线下交付、完成订单、评价 | 展示校园线下面交、订单状态、评价影响信誉 |
| 16-18 分钟 | 智慧清单 | `/smart-list`：一键生成模拟课表/导入课表，展示教材计划和知识推荐 | 展示“课程驱动”的智能推荐，不展示草稿商品 |
| 18-20 分钟 | 知识星河收束 | `/knowledge`：签到、上传资料、下载/收藏、讨论 | 展示知识经验分享和积分机制，形成产品差异化 |

### 展示口径

- 不要说“我们依次看每个页面”，建议说：“我用一个新同学的一天来演示：注册进入平台，找书、买书、卖书、沟通交易，最后把课程资料沉淀到知识星河。”
- 页面可以跳，但逻辑不能断：每次跳转都说明“这一步解决什么用户问题”。
- 专家更关心闭环和异常处理：注册失败/密码错误、缺货登记、订单取消、草稿不进入商品池、资料删除或下架，任选 2-3 个快速展示即可。
- 15-20 分钟不要展开所有筛选项、所有 tab。每个模块只做 1 个代表操作。

### 2.3 市场看板卡片
**内容**：

- 后端：`GET /api/orders?role=all` → `orders.py`

**讲解要点**：订单状态流转可视化，一个订单从创建→沟通→确认见面→完成，各个阶段都会在这里体现

### 2.4 信誉状态卡片 — 信誉机制详解
**显示内容**：
- 当前信誉分与等级
- 已完成交易数量
- 信誉状态摘要

**信誉总分规则**：
`买家评价星级 × 70% + 近30天响应时效 × 30%`

| 分项 | 计算口径 |
|------|----------|
| 买家评价 | 先按“平均星级 ÷ 5 × 100”得到原始分，再按 `(原始分 × 评价数 + 100 × 5) ÷ (评价数 + 5)` 做低样本平滑 |
| 响应时效 | 近 30 天买家发起咨询到卖家首次回复的中位时间；至少 3 次有效回复后参与计分 |

当前版本暂不对取消行为自动判责，因此取消订单不纳入信誉扣分，避免误伤正常协商取消场景。没有有效样本的分项按 100 分中性起步，并在审计日志中标记样本不足。买家评价会做低样本平滑，避免前几笔订单把信誉分拉动得过猛。所有取消仍会生成一条审计快照，但只记录，不扣分。


**信誉等级**：

| 总分 | 等级 |
|------|------|
| 90–100 | 极好 |
| 80–89 | 良好 |
| 70–79 | 一般 |
| 60–69 | 较差 |
| < 60 | 很差 |

**信誉驱动流程 — 如何操作**：
1. 两方完成一笔交易后，订单状态变为 `completed`
2. 买家/卖家在"我的订单"页面点击评价，提交 1–5 分评分
3. 后端 `POST /api/orders/:id/rate`（`orders_reviews.py` 行 59–105）：
   - 保存评分到 `Order.buyer_rating` 或 `Order.seller_rating`
**讲解要点**：信誉是交易评价驱动的动态分数与等级，不是固定值；评价越多，低样本平滑影响越小，结果越接近真实口碑
   - 自动重新计算卖家的三项信誉分
   - 同一事务更新 `User.credit_score`、`User.reputation` 并写入 `credit_audits`
4. 首页信誉卡片通过 `getUserStats()` 读取最新分数与等级
5. 卖家在书籍详情页也会展示信誉等级，买家可据此判断可信度

**数据库中存储**：`users.credit_score`、`users.reputation`；每次计算快照写入 `credit_audits`

**讲解要点**：信誉是交易评价驱动的动态等级，不是固定值；评价越多，等级越有参考价值

### 2.5 心愿提醒
**内容**：展示待确认订单列表
**讲解要点**：及时提醒用户完成交易流程

### 2.6 精选教材

## 🔍 演示提示与常见问题

| 场景 | 处理方式 |
|------|----------|
| 后端未启动 | 页面显示 toast "后端服务暂时不可用" |
| 密码错误登录 | 登录页显示错误提示，停留在登录页 |
| token 过期 | 自动尝试 refresh_token 刷新；失败则跳转登录页 |
| 注册用户名已存在 | 显示"该用户名已存在，请更换其他用户名" |
| 注册邮箱已存在 | 显示"该邮箱已被注册" |
| 搜索无结果 | 显示"没有找到相关书籍" |
| 消息页无会话 | 显示空状态引导文案 |
| 摄像头权限拒绝 | ISBNScanner 提示需要摄像头权限 |
| 没有独立投诉模块 | 用“订单取消 / 缺货登记取消 / 未登录保护 / 下架商品”作为分支流程展示 |
